## a11y verification
- **Measurable row attaches meter role with correct numeric attributes:** The `vitamin_c_mg` test asserts `role="meter"`, `aria-valuenow="33"`, `aria-valuemin="0"`, `aria-valuemax="100"` on `<div data-testid="food-detail-micro-row-vitamin_c_mg">`. The sodium-row test asserts the same on the always-visible default row (`aria-valuenow="35"`).
- **Over-RDA clamp:** The 4600 mg sodium test asserts `aria-valuenow="100"` (clamped) while the DV text suffix reads `200% DV` — proving the user STILL sees overshoot visually while AT receives a valid meter value within `[valuemin, valuemax]`.
- **Non-measurable row omits meter role + DV suffix:** The orphan-key (`omega3_g`) test asserts both `queryByTestId('food-detail-micro-dv-omega3_g')` and `queryByTestId('food-detail-micro-row-omega3_g')` return `null`. The sugar default-row test asserts the same. Confirms a row without an RDA reference does NOT lie to SR users by advertising as a meter.
- **aria-label text:** Each measurable row receives a composed label like `"Vitamin C 30 mg, 33% daily value"` — name + value + DV percent in one phrase, so a screen reader announces the full meter context without the user having to navigate into individual cells.
