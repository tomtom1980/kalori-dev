## Top 10 most-relevant lessons

1. **E2E specs that import `@playwright/test` directly (skipping the project's auth fixture) silently land on `/login` while assertions still pass against the redirect target.** Unit/integration suites mocking auth never expose the gap; phase sweep is where it surfaces. Phase Codex Review or briefing checklists must require auth-fixture import on every E2E spec touching protected routes. (line 30)

2. **[2026-05-07] Orphan-profile / partial-state E2E fixtures: delete the dependent row only, never the parent identity row.** Faking auth.users + valid JWT + missing downstream profile/membership row requires: (a) `signInWithPassword` ephemeral user, (b) capture auth state, (c) service-role `from('<dependent_table>').delete().eq('<fk>', userId)` to drop ONLY the dependent row, (d) yield Page, (e) teardown via `auth.admin.deleteUser` in `try/finally`. Calling `auth.admin.deleteUser` at fixture-time cascades via FK and produces an UNAUTHENTICATED state — wrong fence branch entirely (covers 401-without-JWT path, not 401-with-orphan-JWT path). (line 108)

3. **[2026-05-07] When fixing a status code in shared infrastructure, sweep ALL test files asserting the old code, not just the obvious ones; mock helpers in unrelated suites can hold stale contracts.** Phase A auto-fix sub-agent updated orphan-fence's status code but missed `tests/unit/ai/vn-smoke.test.ts`'s mock helper which short-circuited all 5 VN smoke fixtures with the old code. Pre-push hook caught the regression. Pattern: when a status code/error code/contract value changes in shared infra, run `grep -rn "<old-value>" tests/` BEFORE committing. (line 110)

4. **[2026-05-08] Fixture extensions that paper over a server contract violation MASK a production bug; the real-POST E2E that exposed the bug is the right surface to keep — log a Critical followup, don't silently absorb.** When extending an E2E auth fixture to make a real-POST AC pass (e.g. flipping `timezone` from Asia/Ho_Chi_Minh to UTC to dodge `400 date_in_future`), the test workaround is a productivity fix BUT the underlying server bug exists for production users. Discipline: (a) fixture extension ships in same commit as failing AC, (b) Critical followup filed naming production-impact, (c) once server contract is fixed, fixture flip reverts. Mocked unit tests would have HIDDEN this entirely; the real-POST E2E EXPOSED it — keep real-POST shape, never collapse to mock-and-forget. (line 66)

5. **[2026-05-07] When a server-side cache-invalidation primitive (`revalidatePath`) coexists with a client-side self-hydration codepath, E2E must PROVE the server primitive is load-bearing — not be silently propped up by client polling.** Pattern: attach `page.route('/api/<self-hydrate-route>', route => { calls.push(...); route.continue() })` listener BEFORE the user action. After the action + UI assertion lands, assert `calls.length === 0`. If the listener fires non-zero, emit `console.warn` NOTABLE — server invalidation may be dead code masked by client redundancy. (line 109)

6. **[2026-05-08] 2-round fix circuit breaker semantics: fix the SPEC if the bug is in the spec; HALT if the bug is in production code.** During an E2E task, when Round 1 test failure traces to a real production bug, DON'T silently patch production behavior to satisfy the spec. Pattern: (a) confirm production gap via inspection, (b) refine the test to a smoke-level observable (e.g. coarse-grain assert via toast/server-data round-trip rather than direct predicate assertion), (c) emit `console.warn` flag, (d) file NEW followup with severity Architectural/Critical naming the production gap, (e) surface the gap in evidence narrative. (line 72)

7. **[2026-05-07] Next 16 turbopack dev mode pollutes module state across specs on the same dev server — fenced API routes can return `404 text/html` (route module not compiled) AND adjacent fenced routes can return `200` (fence appears not to fire) after a prior probe spec ran on the server.** Symptom: bundled spec was GREEN against fresh `pnpm dev`, then RED on second invocation when an earlier spec touched related routes. Restarting dev server cleanly restored 401 behavior across ALL fenced routes. Production has no such pollution. Mitigations: (a) Playwright `webServer` config provides fresh server per run; (b) local re-runs may need `pnpm dev` restart between unrelated probe specs; (c) DON'T file as production fence regression — verify on fresh server first. (line 254)

8. **[2026-05-08] E2E discovers mount-lifecycle bugs unit tests miss when listener lifetime is tied to a component subtree that unmounts before the predicate flips.** Unit tests render listener-owning component standalone — listener subscribes, predicate flips, listener fires. Production embeds same component inside state-machine swap (tab/phase transitions where sibling replaces subtree); listener's `useEffect` cleanup unsubscribes BEFORE predicate flip happens elsewhere, and rising edge is missed. Two compatible fixes: (a) relocate listener to a chrome-level component that stays mounted across phase transitions; (b) move state mutation INTO the store action that flips the predicate. (line 71)

9. **[2026-05-08] When evaluating an SLA under parallel test workers, either cap test concurrency to the SLA budget OR evaluate SLA under realistic worker load — never silently relax the SLA cap to absorb worker contention.** A bundled E2E spec running with 4 parallel workers can breach a per-story SLA hard cap (e.g. 1500ms target/3000ms cap) under contention even when per-story spec running solo passes. Pattern: (a) keep per-story SLA cap as load-bearing CI gate, (b) if bundled spec must run with N>1 workers, calibrate cap against measured contention AND add `console.warn`/JSON log for variance, (c) consider running bundled spec with `workers: 1`. (line 74)

10. **[2026-05-08] COEXIST — keep per-story E2E specs alongside a later-added bundled phase E2E spec; never delete per-story specs to "deduplicate."** Per-story specs are RED→GREEN trace artifact of user story's implementation task — `git blame` tells next reader which AC drove which behavior change. Bundled phase specs aggregate phase-gate coverage. Per-story specs may carry stricter SLA caps (3000ms hard cap as load-bearing CI gate); bundled specs may relax SLA caps to absorb parallel-worker contention (5000ms with inline rationale). Rule: when introducing bundled phase E2E later, COEXIST with all per-story specs; never delete to deduplicate. (line 185)
